# 节点介绍本地化设计

## 目标

`Node.vue` 的购买节点介绍弹窗不再展示 `apiNodes()` 返回的 `intro`，而是展示从《节点介绍.docx》整理出的本地多语言 HTML 富文本。

## 数据与映射

- 新建独立的节点介绍数据模块，避免长文本进入 `Node.vue` 或现有通用 locale JSON。
- 每种语言保存两个 HTML 字符串，并保持统一数组顺序：
  - 索引 `0`：Visa卡业务分红节点。
  - 索引 `1`：AI量化手机节点。
- 支持当前语言选择器启用的语言：`en`、`ko`、`ja`、`ru`、`hi`、`ma`、`id`、`vi`、`hk`、`zh`。
- 未知语言回退到简体中文；无效节点索引返回空字符串，避免错误展示另一节点的信息。

## 组件数据流

`Node.vue` 继续使用接口节点数据展示名称、图片、价格、库存和购买所需 ID。新增一个计算属性，读取当前 i18n locale 和 `pickerCurrent`，从本地数据模块取得对应介绍。弹窗的 `v-html` 改为绑定该计算属性，接口中的 `intro` 不再参与渲染。

语言切换后，计算属性随全局 locale 更新，当前弹窗内容无需重新请求接口即可切换语言。

## 富文本结构

文档内容转换为受控的静态 HTML，只使用标题、段落、有序/无序列表和强调标签。内容保留原文的章节、编号、金额、日期及风险提示层级。样式由 `Node.vue` 的弹窗内容容器统一控制，保证长内容可滚动、标题与段落有清晰间距。

## 安全与兼容

HTML 内容全部来自代码仓库中的静态常量，不拼接用户输入或接口数据。数据模块提供明确的语言与索引类型，组件不依赖节点名称匹配，因此接口只需维持现有两个节点的固定顺序。

## 验证

- 为语言回退、两个索引映射和无效索引编写最小自动化测试（若项目现有工具链可直接支持）。
- 运行 TypeScript/Vue 类型检查及生产构建。
- 检查 `Node.vue` 已不再引用 `currentPicker.intro`，并抽查简体中文、繁体中文和英文富文本的章节及关键数字。
